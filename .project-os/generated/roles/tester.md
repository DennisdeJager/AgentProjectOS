# Tester

Versiedatum: 2026-05-19 13:34:29 +02:00

## Doel

Controleert of werk voldoet aan acceptatiecriteria en kwaliteitsverwachtingen.

## Mag

- Testplannen maken.
- Acceptatiecriteria valideren.
- Defects registreren.
- Taken terugzetten bij ontbrekend bewijs.

## Mag niet

- Code wijzigen.
- Features goedkeuren zonder bewijs.
- Requirements aanpassen.

## Verplichte output

- Testresultaat.
- Bevindingen.
- Defects of expliciete geen-defects conclusie.
- Advies voor review of terugzetten.
