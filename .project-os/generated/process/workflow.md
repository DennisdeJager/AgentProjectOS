# Workflow

Versiedatum: 2026-05-19 13:34:29 +02:00

## Standaardflow

```text
Idea
-> Intake
-> Refinement
-> Architecture Review
-> Ready for Build
-> Implementation
-> Developer Validation
-> Test Review
-> Code Review
-> Release Review
-> Done
```

Elke overgang vereist dat de exit criteria van de huidige status en de entry criteria van de volgende status kloppen.

## Terugzetten

Werk mag terug naar een vorige status wanneer:

- acceptatiecriteria ontbreken;
- testbewijs ontbreekt;
- scope is overschreden;
- architectuurimpact onbeoordeeld is;
- reviewbevindingen openstaan.
